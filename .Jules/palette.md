## 2024-03-24 - Accessible Inline Form Validation

**Learning:** Native HTML5 `setCustomValidity` combined with visual error text (e.g., toggling `display: block` on a `<small>` tag) is insufficient for screen readers on dynamically validated inputs. Screen readers need a direct programmatic association between the input and the error text, and they need to be notified when the error state changes dynamically during typing.

**Action:** When implementing inline form validation triggered by JavaScript events (like `input` or `change`), always:
1. Link the error message to the input using `aria-describedby`.
2. Add `aria-live="polite"` to the container holding the error message so dynamic visibility changes are announced.
3. Programmatically toggle the `aria-invalid` attribute on the input itself (`"true"` when invalid, `"false"` or remove when valid).
4. Avoid relying on implicit global variables generated by HTML `id` attributes to manipulate DOM elements in vanilla JS; always explicitly select them with `document.getElementById()`.
